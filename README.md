# Project-1-LightLink

基于OpenCV和FFmpeg的视频编解码项目，用于计算机网络课程实验。

## 项目状态

- **当前阶段**：编码和解码功能开发完成，调试优化中
- **依赖项**：需要用户自行配置 OpenCV 和 FFmpeg

## 目录结构

```
Project-1-LightLink/
├── bin/                          # 运行时目录
│   ├── Project-1-LightLink.exe  # 编译后生成
│   ├── ffmpeg/                  # FFmpeg工具
│   ├── inputImg/                # 输入图片目录
│   ├── inputVideo/              # 输入视频目录
│   ├── mid_operation/            # 中间处理目录
│   └── output/                  # 输出文件目录
├── src/                          # 源代码和依赖
│   └── opencv/                  # OpenCV头文件
├── asset/                        # 测试资源
├── Project-1-LightLink/          # VS项目文件
├── README.md
└── Project-1-LightLink.slnx
```

## 环境配置（拉取项目后必须操作）

### 第一步：下载依赖库

#### 1. 下载 OpenCV（版本 4.12.0）

- 官网：https://opencv.org/releases/
- 选择 Windows 版本
- 下载后解压到项目 `src/` 目录下

#### 2. 下载 FFmpeg（版本 8.0.1）

- 官网：https://ffmpeg.org/download.html
- 选择 "essentials" 版本 for Windows
- 下载后解压，将 `bin` 文件夹复制到 `bin/ffmpeg/` 目录

### 第二步：配置 Visual Studio 项目

1. 用 VS2022 打开 `Project-1-LightLink.slnx`
2. 右键项目 → 属性
3. 配置以下路径（Debug | x64）：

| 配置项 | 路径 |
|--------|------|
| 包含目录 | `$(SolutionDir)src\opencv\build\include` |
| 库目录 | `$(SolutionDir)src\opencv\build\x64\vc16\lib` |
| 附加依赖项 | `opencv_world4120d.lib` |
| 输出目录 | `$(SolutionDir)bin\` |

### 第三步：编译运行

1. 生成 → 生成解决方案
2. 生成的 exe 会在 `bin/` 目录

## 使用说明

### 编译宏定义

在 `myTool.cpp` 中选择模式：
```cpp
#define _ENCODE  // 编码模式
// 或
#define _DECODE  // 解码模式
```

### 编码模式

```bash
Project-1-LightLink.exe [输入图片] [输出视频]
```

示例：
```bash
Project-1-LightLink.exe asset/test.jpg output/test.mp4
```

### 解码模式

```bash
Project-1-LightLink.exe [输入视频] [输出图片]
```

示例：
```bash
Project-1-LightLink.exe output/test.mp4 output/result.png
```

## 编码方案说明

### 技术参数

| 参数 | 值 |
|------|-----|
| 每帧数据量 | 1242 字节 |
| 二维码尺寸 | 108 × 108 像素 |
| 信息区数量 | 10 个 |
| 总数据容量 | 1420 帧 ≈ 1.76 MB |

### 帧结构

```
┌─────────────────────────────────────┐
│ 定位点 (左上、左下、右上)            │
├─────────────────────────────────────┤
│ 帧标志位 (col=87, rows=2-5)        │
│ 帧编号   (col=88, rows=2-17)       │
│ 校验码   (col=89, rows=2-17)       │
├─────────────────────────────────────┤
│ 数据长度 (col=87, rows=6-17)        │
├─────────────────────────────────────┤
│ 信息区 0-9                          │
└─────────────────────────────────────┘
```

### 信息区布局

| 区域 | 位置 (row, col) | 尺寸 (rows×cols) | 容量 (字节) |
|------|-----------------|------------------|-------------|
| 0 | 2, 18 | 16×34 | 68 |
| 1 | 2, 52 | 16×35 | 70 |
| 2 | 18, 2 | 36×52 | 234 |
| 3 | 18, 54 | 36×52 | 234 |
| 4 | 54, 2 | 36×52 | 234 |
| 5 | 54, 54 | 36×52 | 234 |
| 6 | 90, 18 | 16×36 | 72 |
| 7 | 90, 54 | 16×36 | 72 |
| 8 | 90, 90 | 8×16 | 16 |
| 9 | 98, 90 | 8×8 | 8 |

### 帧标志位定义

| 标志 | 位模式 (b0 b1 b2 b3) | 说明 |
|------|----------------------|------|
| 起始帧 | 黑白白黑 (B W W B) | 第一帧 |
| 中间帧 | 黑白黑白 (B W B W) | 普通帧 |
| 终止帧 | 白黑黑白 (W B B W) | 最后一帧 |
| 单帧 | 白黑白黑 (W B W B) | 既是起始也是终止 |

### 特殊处理

1. **随机化**：使用可逆的XOR操作对数据进行混淆
2. **校验码**：CRC16校验，确保数据完整性
3. **JPEG容忍**：使用灰度阈值(128)判断黑白，应对压缩损失
4. **帧放大**：编码端输出108×108帧，FFmpeg放大至1080×1080

## 解码方案说明

### 与编码方案的对应关系

1. **帧尺寸处理**：
   - 编码：生成 108×108 帧 → ffmpeg 放大到 1080×1080 → 视频
   - 解码：视频拆帧 1080×1080 → resize 到 108×108 → 解码
   - **注意**：当前每10×10像素对应1个bit位

2. **关键参数**：
   - `FrameSize = 108`
   - `BytesPerFrame = 1242`
   - `LEN_MIN = {68, 70, 234, 234, 234, 234, 72, 72, 16, 8}`

3. **像素判断**：
   - 使用灰度阈值：< 128 为黑色，>= 128 为白色
   - 解决 JPEG 压缩导致的颜色偏差问题

4. **原始图片尺寸**：
   - 编码输入：1024 × 574 像素
   - 解码输出：1024 × 574 像素
